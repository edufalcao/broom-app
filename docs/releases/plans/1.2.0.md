## Overview

v1.2.0 adds a new Large File Finder feature, expands the cleaner with Docker and Homebrew cleanup, completes the remaining polish items from the original implementation plan (Step 14), and improves the orphan leftover UI.

---

## Feature 1: Large File Finder

**Priority:** P0
**Sidebar:** New tab — "Large Files" (SF Symbol: `doc.badge.arrow.up`)

A new section that scans the user's home directory for files exceeding a configurable size threshold, sorted by size descending.

### Behavior

- Scan `~/` recursively for files > 100 MB (configurable in Settings)
- Skip `.Trash`, `Library/Caches`, `.git` directories, and system paths
- Display results as a flat, sortable list: file name, path, size, last modified
- Each item has a checkbox for selective deletion
- "Move to Trash" button for selected items
- No items pre-selected — user must explicitly choose

### UI

- Sidebar item: "Large Files" with `doc.badge.arrow.up` icon
- States: idle (with "Scan" button), scanning (progress), results (list), done
- Results list columns: checkbox, file icon, name, path (truncated), size, modified date
- Sort by: size (default), name, modified date
- Filter: minimum size picker (100 MB, 250 MB, 500 MB, 1 GB)

### Implementation

| File | Description |
|------|-------------|
| `Models/LargeFile.swift` | Model: path, name, size, modifiedDate, isSelected |
| `Services/LargeFileScanner.swift` | Actor: recursive scan with size threshold, progress reporting |
| `ViewModels/LargeFilesViewModel.swift` | @MainActor @Observable: scan state machine, selection, sort |
| `Views/LargeFiles/LargeFilesView.swift` | Main view with state routing (idle/scanning/results) |
| `Views/LargeFiles/LargeFileRowView.swift` | Single row: checkbox, icon, name, path, size |
| `BroomTests/LargeFileScannerTests.swift` | Test with temp directory and known file sizes |

### Settings Addition

Add to Cleaning settings tab:
- Minimum large file size: picker (100 MB, 250 MB, 500 MB, 1 GB), default 100 MB

## Feature 2: Docker Cleanup

**Priority:** P1
**Location:** New category row inside Clean scan results

Detect and report Docker disk usage. Only shown if Docker is installed.

### Scan Targets

| Path | Description |
|------|-------------|
| `~/Library/Containers/com.docker.docker/Data/vms/` | Docker VM disk images |
| `~/.docker/` | Docker config and local state |

### Implementation

- Add `scanDocker()` method to `FileScanner`
- Add Docker paths to `Constants.swift`
- Returns `CleanCategory?` (nil if Docker not installed)
- Category name: "Docker Data", icon: `cube.box`
- Add to `ScanPhase` enum

## Feature 3: Homebrew Cleanup

**Priority:** P1
**Location:** New category row inside Clean scan results

Report old Homebrew formula downloads and stale caches. Only shown if Homebrew is installed.

### Scan Targets

| Path | Description |
|------|-------------|
| `~/Library/Caches/Homebrew/` | Downloaded bottles (already scanned as dev cache) |
| `/opt/homebrew/Cellar/` or `/usr/local/Cellar/` | Old formula versions (report only, not auto-selected) |
| `$(brew --prefix)/share/doc/` | Documentation files |

### Implementation

- Add `scanHomebrewExtended()` method to `FileScanner`
- Detect Homebrew prefix dynamically (`/opt/homebrew` on ARM, `/usr/local` on Intel)
- Category name: "Homebrew", icon: `mug`
- Items default to unselected (old formula versions could break things)
- Add to `ScanPhase` enum

## Feature 6: Orphan UI Hierarchy

**Priority:** P1
**Location:** `ScanResultsView.swift` orphan section

Improve visual distinction between parent orphan apps and their child file locations.

### Current Problem

Parent orphan rows and expanded child location rows have the same indentation, padding, and visual weight. There's no clear nesting hierarchy.

### Solution

- **Indent** child locations with additional left padding (~24px)
- **Background** group children under a subtle rounded background (`quaternarySystemFill`)
- **Icon** differentiation: parent uses `archivebox` (filled), children use `folder` (outline) at smaller size
- **Typography**: children use `.callout` font vs parent `.body`
- **Connector**: subtle left border or indent line connecting children to parent

### Files to Change

| File | Change |
|------|--------|
| `Views/Cleaner/ScanResultsView.swift` | Restyle `orphanLocations()` with indentation and background |
