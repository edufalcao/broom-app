# Broom v1.3.0 — Product Parity and Trust Pass

> **Date:** 2026-03-15
> **Status:** Implemented on `main`
> **Base:** v1.2.1

---

## Summary

v1.3.0 closes the highest-value gaps between the docs and the shipped app behavior. The release focuses on trust, parity, and macOS-native polish: consistent defaults, broader cleaner coverage, faster scanning, more complete app discovery, safer uninstall handling, and evergreen documentation that matches the current implementation.

## Goals

- Make documented defaults match runtime behavior.
- Expand cleaner coverage where the docs already implied support.
- Improve perceived quality with concurrent scanning and deterministic output.
- Tighten uninstall and inventory flows so the app feels first-class on macOS.
- Leave the repo with evergreen product and engineering docs that describe the current app truthfully.

## Scope

### 1. Cleaner Defaults and Coverage

- Unify temporary-file cleanup defaults at 7 days across runtime, settings, README, PRD, and release notes.
- Make notifications default to enabled on first launch, not only after the user toggles the setting.
- Add a Downloads awareness category to the cleaner:
  - Visible only when non-empty.
  - Shown unselected by default.
  - Informational unless the user explicitly selects it.

### 2. Scanner Performance and Correctness

- Run cleaner categories concurrently with `TaskGroup`.
- Preserve deterministic category ordering in final UI results.
- Keep progress reporting stable enough for the existing SwiftUI scan states.
- Ensure categories with `defaultSelected = false` also initialize their child items as unselected.

### 3. Inventory and Uninstall Improvements

- Supplement installed-app discovery with Spotlight for `.app` bundles outside `/Applications` and `~/Applications`.
- Keep launch agents and daemons in uninstall previews.
- Add a force-quit fallback in the uninstall flow when graceful termination fails.
- Narrow docs so they only promise what the app actually removes today.

### 4. Documentation and Release Metadata

- Refresh README, PRD, architecture, implementation-plan status, changelog, and in-app release notes.
- Bump the app to `1.3.0`.
- Keep the plan archived in `docs/releases/plans/` with an implemented status once the work lands.

## Acceptance Criteria

- Temp-file age defaults are 7 days everywhere.
- Notification delivery honors a default-enabled preference on a clean first launch.
- Downloads appears as an awareness-only cleaner category and starts unchecked.
- Cleaner scan categories run concurrently while result ordering stays stable.
- App inventory can discover Spotlight-only apps in non-standard locations.
- Running-app uninstall flow offers a force-quit path after graceful termination fails.
- Evergreen docs describe the current sidebar, defaults, test counts, and shipped feature set.

## Verification

- `xcodebuild test -project Broom.xcodeproj -scheme Broom -configuration Debug`
- Expected result after implementation: 72 tests across 21 suites passing.

## Outcome

This plan shipped as `v1.3.0` on `main`, with the app config, tests, docs, and release metadata updated together.
