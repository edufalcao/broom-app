## Broom v1.1.0 - Cleaner and Uninstaller Update

**Broom 1.1.0** builds on the initial release by wiring orphan cleanup directly into scan results, making cleaning preferences affect runtime behavior, expanding the app uninstaller, and improving the Apps split view when the window is inactive. This release also adds broader automated coverage across scanner, orphan detection, settings, cleaning, inventory, and uninstall flows, while keeping the app local-first, transparent, and recoverable by default.

---

### System Cleaner
- Orphaned app leftovers are now detected and merged into the main scan results flow
- Cleaning settings now control developer-cache scanning, .DS_Store scanning, temp-file age filtering, and delete behavior
- Running apps are detected before cleaning so active-app caches can be skipped or cleaned intentionally

### App Leftover Cleanup
- Orphaned files now surface as part of the standard cleaner workflow instead of remaining disconnected from scans
- Safe-list rules and protected-path exclusions continue to apply when leftover files are evaluated
- Confidence-based orphan grouping is now backed by the real scan pipeline

### App Uninstaller
- Dragged `.app` bundles now open an uninstall preview even when they were not already indexed
- App sizes include associated files, so sort-by-size reflects the full uninstall footprint
- Uninstall previews now support per-file selection plus Trash or permanent-delete choices
- Associated-file discovery now covers more uninstall targets, including launch agents and launch daemons

### Settings
- Cleaning preferences now apply at runtime instead of acting like UI-only toggles
- The About tab now includes in-app release notes for the current version
- Version and changelog metadata were updated for the 1.1.0 release

### Safety
- Running-app checks guard cache cleaning before deletion starts
- Trash remains the default removal path, with permanent delete still available when explicitly chosen
- Uninstall previews let you review and deselect files before removal

### Quality
- Added broader automated coverage for scanner, orphan detection, settings, cleaning, inventory, and uninstall flows
- Current verification passed with 39 tests across 15 suites
