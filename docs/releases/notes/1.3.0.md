## Broom v1.3.0 — Product Parity and Trust Pass

**Broom 1.3.0** closes the highest-value gaps between the docs and the shipped app: cleaner defaults now match runtime behavior, scans run concurrently with stable ordering, app discovery covers non-standard locations better, and the uninstall flow handles stubborn running apps more gracefully.

---

### System Cleaner
- Temporary-file cleanup now defaults to 7 days consistently across runtime, settings, and docs
- Notifications now default to enabled on first launch instead of silently staying off until toggled
- Downloads appears as an awareness-only category and starts unselected
- Cleaner categories now scan concurrently while preserving deterministic result ordering
- Docker and Homebrew scan coverage now respects injected test locations reliably

### App Leftover Cleanup
- Installed-app inventory now supplements `/Applications` and `~/Applications` with Spotlight-discovered apps in non-standard locations
- Orphan confidence scoring keeps receipt and Spotlight signals as confidence inputs without suppressing valid leftover candidates

### App Uninstaller
- Running-app uninstall flow now offers a force-quit fallback when a normal terminate request fails
- Launch agent and daemon discovery remains part of uninstall previews
- Dropped `.app` bundles continue to route directly into uninstall preparation

### Settings
- Cleaning and General settings now use the same shared defaults as the runtime services
- About now includes a GitHub repository link, author credit, and updated in-app release notes

### Documentation
- README, PRD, architecture, implementation-plan status, and release plan docs now match the current sidebar, defaults, and shipped feature set
- Added the archived release plan for `1.3.0` under `docs/releases/plans/`

### Quality
- 72 tests across 21 suites
- Added coverage for notification defaults, Downloads awareness, parallel scan ordering, Spotlight app inventory, and force-quit uninstall fallback
