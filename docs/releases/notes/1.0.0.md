## Broom v1.0.0 — Initial Public Baseline

**Broom 1.0.0** is the current public baseline release: a native macOS cleanup app that combines a full system cleaner, orphaned-file detection, app uninstall previews, and a dedicated large-file finder, while keeping every destructive action reviewable and Trash-first by default.

---

### System Cleaner
- Scan and remove system caches, browser caches (Chrome, Firefox, Safari, Arc, Brave, Edge), system logs, crash reports, temporary files, Xcode data, developer caches, Docker data, Homebrew data, mail downloads, and .DS_Store files
- Downloads appears as an awareness-only category and starts unselected
- Temporary-file cleanup defaults to 7 days
- Cleaner categories scan concurrently while preserving stable result ordering

### App Leftover Cleanup
- Detect orphaned files left behind by uninstalled apps
- Confidence scoring (high/medium/low) helps judge what is safe to remove
- Leftovers appear directly in the main cleaner results instead of a disconnected workflow

### App Uninstaller
- Browse installed apps with icons, sizes, and last-used dates
- See associated files across Library directories, plus launch agents and launch daemons
- Drag-and-drop a `.app` onto the window to open its uninstall preview
- Running-app uninstall flow supports graceful quit and force-quit fallback
- Spotlight-supplemented inventory helps discover apps outside standard application folders

### Large File Finder
- Scan the home directory for files above 100 MB, 250 MB, 500 MB, or 1 GB
- Sort by size, name, or modified date
- Reveal results in Finder before removal
- Move selected large files to Trash

### Settings & Safety
- Launch at login, delete method, running-app behavior, temp-file age, and safe list settings
- Native macOS Settings window with GitHub and personal-site links in About
- Full preview before deletion, Trash-first defaults, confirmation dialogs, and protected-path exclusions

### Quality
- 72 tests across 21 suites
- MVVM + service-protocol architecture with actor-isolated file-system work
